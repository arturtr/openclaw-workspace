# Родительский рост: дизайн системы

> Спек: пошаговый план развития родительских навыков Артура и Юли.
> Создано: 27.03.2026

## Контекст

Существует детальный анализ родительства по 4 системам (астрология, Ба Цзы, Human Design, MBTI) в `family/parenting-analysis.md`. 12+ точек роста определены, но не превращены в практику. Файл не интегрирован с агентами — лежит мёртвым справочником.

Цель: превратить анализ в живую систему развития — бесконечную игру с углублением.

## Решения

| Элемент | Решение |
|---------|---------|
| Аудитория | Оба: Артур и Юля |
| Формат | Челлендж-карта (на спринт) + ежедневный трекер привычек |
| Спринт | 2 недели, 1 точка роста → 2-3 личных микро-практики + 1 совместная |
| Пилот | 3 спринта (6 недель), потом решение о продолжении |
| Длительность | Бесконечная игра — углубление и расширение |
| Агент | Kuzya ведёт обоих |
| Трекинг | Singularity — привычки на 14 дней |
| Доставка Юле | Решается при актуализации с Юлей |
| Первый шаг | Разговор с Юлей по гайду актуализации |

## Структура файлов

### `family/parenting-catalog.md` — банк точек роста и практик

Растёт бесконечно. Источник правды «что можно делать».

Содержит:
- **Точки роста** — таблицы по трём группам (Артур / Юля / Вместе). Каждая точка пронумерована (А1, Ю2, В3) для ссылок. Статусы: не начато / в работе / проработано.
- **Банк практик** — карточки, привязанные к точкам роста. Каждая карточка: кто, действие, время, сигнал успеха.
- **Архив инсайтов** — что обнаружили в процессе. Пополняется после каждого ретро.

Источник первоначальных точек: `family/parenting-analysis.md` (раздел «Точки роста» + «Сложная семья»). Актуализируются с Юлей перед запуском.

### `family/parenting-sprint.md` — текущий спринт + лог

Источник правды «что делаем сейчас».

Содержит:
- **Текущий спринт** — фокусная точка роста, период, 3 практики (Артур / Юля / вместе), трекинг по дням, заметки по ходу.
- **Лог прошлых спринтов** — по каждому: что взяли, что сработало, что нет, инсайт в каталог, следующий фокус.

## Механика спринта

### Ритм (2 недели)

**Старт спринта (понедельник, неделя 1):**
- Kuzya предлагает фокус из каталога (следующая по приоритету точка роста)
- Артур подтверждает (или меняет)
- Kuzya создаёт привычки в Singularity на 14 дней
- Kuzya обновляет `parenting-sprint.md`

**Неделя 1 — проба:**
- Ежедневно: практики по 3-5 мин каждая
- Singularity трекает выполнение
- Kuzya не пушит, но напоминает по запросу

**Неделя 2 — углубление:**
- Продолжаем то, что зашло
- Отбрасываем то, что не работает
- Заметки по ходу — Kuzya записывает со слов

**Ретро (воскресенье, неделя 2):**
- Kuzya инициирует: «Как прошёл спринт?»
- Записывает в лог спринта
- Переносит инсайты в каталог
- Завершает привычки текущего спринта в Singularity

## Актуализация с Юлей

Перед запуском — разговор по гайду. Не урок, а разговор на равных. Юля — соавтор.

### Гайд (`family/parenting-review-guide.md`)

**По каждой из 12 точек роста:**
- Актуально? (да / частично / нет)
- Что изменилось с марта?
- Приоритет: 1 = горит, 2 = важно, 3 = когда-нибудь

**Открытые вопросы:**
- Что раздражает прямо сейчас? (новые точки)
- Что уже получается хорошо? (убрать из фокуса)
- С какой темы начать пилот?
- Как доставлять практики Юле? (через Kuzya / Артур передаёт / другой вариант)

**Результат:** заполненный каталог + порядок спринтов на пилот.

## Интеграция с агентами

### Kuzya
- `parenting-catalog.md` и `parenting-sprint.md` добавляются в тематическую память (MEMORY.md)
- Грузит по контексту (тема детей/воспитания), не каждую сессию
- Инициирует ретро в конце спринта
- Предлагает фокус нового спринта
- Записывает инсайты в каталог

### Ember
- Не ведёт план — зона Kuzya
- Если Юля упоминает воспитание — Ember может прочитать `parenting-sprint.md` для контекста
- Ссылка на файл добавляется в `agents/ember.md` как опциональная

### Singularity (Артур)
- Проект «Родительство»
- На каждый спринт: 2-3 привычки (повторяющиеся, 14 дней)
- Kuzya создаёт при старте, завершает при ретро

## Пилот: 3 спринта (6 недель)

**Цель пилота:** проверить формат — ритм, практики, трекинг.

**Критерии успеха:**
- Хотя бы 1 практика из каждого спринта прижилась
- Оба участника дошли до ретро хотя бы 2 из 3 раз
- Есть хотя бы 3 инсайта в архиве каталога
- После пилота оба хотят продолжать

**После пилота:** решение — продолжать / менять формат / менять ритм.

## Порядок реализации

1. Создать `family/parenting-review-guide.md` — гайд для разговора с Юлей
2. Провести разговор с Юлей, актуализировать точки роста

4. Создать `family/parenting-sprint.md` — пустой шаблон

7. Обновить `MEMORY.md` — добавить тематические файлы
8. Добавить ссылку в `agents/ember.md` — опциональный доступ к спринту
9. Создать проект «Родительство» в Singularity
10. Запустить первый спринт
