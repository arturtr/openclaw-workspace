# Как организовать рабочее пространство в Singularity App
### Система Юли Трофимовой — коуча, блогера, мамы и исследовательницы

*Версия: март 2026*

---

> Этот гайд написан не как инструкция к приложению. Это описание живой системы, которая реально работает — и которую может повторить любой человек за один вечер.

---

## 1. Философия: зачем вообще нужна система

### Не задачник, а вторая голова

Большинство людей используют таск-менеджер как список дел: записал — выполнил — вычеркнул. Это работает для простых задач, но ломается, когда жизнь сложная.

Юля использует Singularity иначе. Для неё это **вторая голова** — место, где хранятся не просто задачи, но и контекст, исследования, решения, ссылки, заметки. Всё, что иначе крутилось бы в голове и создавало фоновый шум.

Когда система хранит контекст — голова становится свободнее. Можно думать, а не помнить.

### Принцип «Меньше давления, больше глубины»

Обычные системы создают давление: список растёт, задачи просрочиваются, возникает чувство вины. Юля строила систему с обратным эффектом.

**Меньше давления** — это значит:
- Задачи не кричат «выполни меня!» — они ждут своего времени в правильных колонках
- «Когда-нибудь» — это не мусорка, это законное место для идей без дедлайна
- Еженедельный обзор снимает тревогу «а вдруг я что-то забыла»

**Больше глубины** — это значит:
- К каждой задаче есть контекст: зачем, как, что уже сделано
- Блокноты хранят знания, а не просто напоминания
- Исследования сохраняются — и к ним можно вернуться

### Все проекты начинаем с исследования

Прежде чем делать — нужно понять. Это принцип, который пронизывает всю систему.

Новый раздел жизни? Сначала — исследование. Новый проект? Сначала — собрать информацию. Только после этого — задачи.

Поэтому в каждом разделе есть блокнот «Исследования». Это не архив — это живой фундамент, на котором строятся решения.

## 2. Архитектура workspace: три уровня

Вся система строится на трёх уровнях. Как матрёшка — каждый уровень вложен в предыдущий.

```
Раздел (область жизни)
  └── 📋 Задачи [Раздел]  ← единственное место для задач
  └── 📓 Блокнот 1        ← заметки, материалы, ссылки
  └── 📓 Блокнот 2
        └── Заметка 1
        └── Заметка 2
```

### Уровень 1: Разделы = области жизни

Разделы — это верхний уровень. Каждый раздел = одна сфера жизни или крупный проект.

У Юли их семь:

| Раздел | Что это |
|--------|---------|
| **Homo Divinus** | Личный бренд, блог, исследовательская работа |
| **Коучинг** | Клиентская работа, программы, инструменты |
| **Домоведение** | Всё про дом: быт, кухня, хозяйство |
| **Обучение** | Курсы, книги, новые навыки |
| **Здоровье** | Тело, врачи, процедуры, самочувствие |
| **Личное** | Рефлексия, дневник, личное развитие |
| **#яверю** | Духовная жизнь, вера, молитвенные намерения |

Разделы не меняются часто. Это стабильные «комнаты» в доме.

**Как выбрать свои разделы?**
Запиши все сферы своей жизни на листке. Объедини похожие. Оставь 5–8 крупных. Больше — уже путаница.

---

### Уровень 2: Подпроекты двух типов

Внутри каждого раздела — подпроекты. И здесь ключевое правило системы:

**Подпроекты бывают ТОЛЬКО двух типов — никак иначе.**

#### 📋 Задачи [Раздел] — единственное место для задач

Это специальный подпроект с эмодзи 📋. Его название всегда: «Задачи [Название раздела]».

Например:
- 📋 Задачи [Homo Divinus]
- 📋 Задачи [Коучинг]
- 📋 Задачи [Домоведение]

**Зачем это правило?**

Потому что в каждом разделе — одна точка входа для всех дел. Не надо искать задачи по разным местам. Открываешь канбан — видишь всё.

Все задачи раздела живут здесь. Без исключений.

#### 📓 Блокноты — вторая память

Блокноты — это НЕ задачи. Это место для знаний, материалов, исследований, ссылок.

Примеры блокнотов у Юли:

В разделе **Homo Divinus**:
- 📓 Исследования
- 📓 Сайт
- 📓 Кухня с душой (тема блога)
- 📓 Выпуск 1, Выпуск 2... (эпизоды подкаста/видео)

В разделе **Личное**:
- 📓 2026 / Январь
- 📓 2026 / Февраль
- 📓 2026 / Март

В разделе **Здоровье**:
- 📓 Анализы и результаты
- 📓 Исследования

Блокнот — это контейнер. Внутри него — заметки (об этом в разделе 5).

---

### Уровень 3: Задачи внутри 📋

Задачи живут только в 📋-подпроектах. Они видны на канбане и имеют структуру (об этом в разделе 4).

---

### Канбан: четыре колонки

Канбан — это доска, где видны все задачи из всех разделов одновременно.

Юля использует четыре колонки:

```
Новое → План → В работе → Готово
```

| Колонка | Что здесь |
|---------|-----------|
| **Новое** | Задача только что появилась. Ещё не решила, когда займусь |
| **План** | Собираюсь сделать в ближайшее время. Взяла в план |
| **В работе** | Делаю прямо сейчас. Максимум 3–5 задач |
| **Готово** | Выполнено ✓ |
| **Когда-нибудь** | Встроенная категория Singularity. Идеи без срока |

**Главное правило канбана:** «В работе» — это не «взяла, но не делаю». Это то, что открыто прямо сейчас.

«Когда-нибудь» — это не мусорка. Это хорошая полка для идей, до которых руки дойдут потом. Их не нужно выбрасывать — нужно честно убрать с глаз долой.

## 3. Система тегов: навигация без хаоса

Теги — это способ найти задачу из любого места. Но теги работают только если их мало и они структурированы.

### Правило Юли: не больше 20 тегов на всю жизнь

Больше 20 тегов — уже путаница. Когда тегов слишком много, ты перестаёшь ими пользоваться.

### Три категории тегов

**1. Главный тег = название раздела**

Каждая задача получает один главный тег — тот раздел, к которому относится:
- `Homo Divinus`
- `Коучинг`
- `Домоведение`
- `Обучение`
- `Здоровье`
- `Личное`
- `#яверю`

Этот тег обязателен. Всегда. Он позволяет фильтровать канбан по разделу.

**2. Дополнительные теги = тема внутри раздела**

1–2 дополнительных тега уточняют тему:
- `HD-site` — задачи по сайту в разделе Homo Divinus
- `HD Дети` — контент про детское развитие
- `Юрист` — юридические вопросы
- `Курсы` — обучение
- `Кухня` — кулинарные задачи

**3. Теги-люди**

Если задача связана с конкретным человеком:
- `Юля` — моя задача / мне нужно
- `Артур` — задача для Артура или совместная
- `Маша` — подрядчик, помощник
- `Алена` — другой человек из команды

### Правило тегирования

На одну задачу: **1 главный тег + максимум 1–2 дополнительных**.

Пример:
- Задача «Написать статью про детское питание» → теги: `Homo Divinus`, `HD Дети`
- Задача «Починить кран на кухне» → теги: `Домоведение`, `Кухня`
- Задача «Записаться к эндокринологу» → теги: `Здоровье`

## 4. Анатомия задачи: из чего состоит хорошая задача

Плохая задача: «Написать статью»
Хорошая задача: «📝 Написать статью про детское питание [ссылка на черновик]»

Разница в том, что хорошая задача содержит достаточно контекста, чтобы вернуться к ней через неделю и сразу понять, что делать.

### Эмодзи в заголовке

Эмодзи — это визуальный маркер типа задачи. Одного взгляда достаточно, чтобы понять характер дела:

| Эмодзи | Тип задачи |
|--------|-----------|
| 📝 | Написать текст, контент |
| 🏕 | Мероприятие, выезд, событие |
| 💎 | Важное, ключевое |
| 🩺 | Здоровье, медицина |
| 💰 | Финансы, деньги |
| 📷 | Фото, видео, съёмка |
| 🎓 | Учёба, обучение |
| 🎨 | Дизайн, творчество |
| 📚 | Книга, чтение |
| 📄 | Документ, юридическое |
| 🌱 | Развитие, рост |

### Описание задачи

Описание — это контекст. Здесь:
- Что именно нужно сделать (конкретнее, чем в заголовке)
- Ссылки на Dropbox (не markdown-форматирование — просто ссылка для клика)
- Кто вовлечён, откуда взялась задача
- Результат, который нужен

Пример хорошего описания:
```
Черновик: https://dropbox.com/...
Источник: разговор с Машей 14 марта
Нужно: 1500 слов, тон — тёплый и практичный
Дедлайн мягкий — до конца марта
```

### Чек-лист: всегда

Чек-лист — обязательная часть любой нетривиальной задачи. Даже если пунктов три.

Почему? Потому что чек-лист помогает **оценить сложность задачи** до того, как ты её начала. «Написать статью» кажется простым, пока не напишешь: собрать источники, сделать структуру, написать черновик, отредактировать, подобрать картинки, опубликовать.

Вот это уже 6 шагов — и ты понимаешь, что статья займёт не час, а три.

### Дата

Ставится только если есть реальный дедлайн. Не нужно ставить дату «просто так» — это создаёт искусственное давление.

### Приоритет

Singularity использует числовую шкалу (важно: **цифры работают наоборот** от интуитивного!):
- **0** = высокий приоритет 🔴
- **1** = обычный приоритет
- **2** = низкий приоритет 🔵

## 5. Блокноты — вторая память

Если задачи — это «что сделать», то блокноты — это «что знаю».

Блокнот не требует действий. Он хранит знание.

### Структура блокнота

**Описание проекта** (первое, что пишешь при создании блокнота):
- Что это такое?
- Зачем он существует?
- Какие материалы здесь хранятся?

Пример описания блокнота «Исследования»:
> Здесь хранятся все исследования по проекту Homo Divinus. Каждое исследование — отдельная заметка. Формат: дата + тема + саммари + ссылка на полный материал в Dropbox.

### Заметки внутри блокнота: атомарный принцип

Одна заметка = одна единица знания.

Не нужно складывать всё в одну большую заметку. Лучше много маленьких — их легче найти, перечитать и использовать.

**Формат заметки:**
```
**Дата:** 15 марта 2026
**Тема:** [о чём эта заметка]
**Саммари:** [2–3 предложения — суть]
**Контекст:** [откуда взялось, зачем важно]
**Ссылка:** https://dropbox.com/... [если есть полный материал]
```

### Примеры живых блокнотов

**📓 Исследования (в разделе Homo Divinus)**

Каждый раз, когда AI-агент проводит исследование — результат сохраняется сюда. Не в задачу, не в голову — в блокнот. С датой и саммари. И ссылкой на полный отчёт в Dropbox.

Через полгода здесь накапливается библиотека знаний по всем темам проекта.

**📓 Личное / 2026 / Март**

Личный дневник в формате блокнота. Внутри:
- Заметки по неделям: что происходило, что заметила
- Итог месяца: паттерны, выводы, рекомендации себе
- «Покаяние» — честный взгляд на то, что не получилось и почему

Это не задачи. Это рефлексия. Она живёт в блокноте, а не в канбане.

**📓 Кухня с душой**

Концепции, рецепты, идеи для контента на тему кулинарии. Каждый рецепт — заметка. Каждая идея для поста — заметка. Потом из блокнота рождаются задачи (но не автоматически — только когда принято решение делать).

## 6. Еженедельный обзор: пятница как ритуал

Система не живёт без обслуживания. Еженедельный обзор — это 30–60 минут в пятницу, которые держат всё в порядке.

### Что делать на обзоре

**Шаг 1. Просмотреть все блокноты**
Открыть каждый активный блокнот и пробежать глазами последние заметки. Что там подвисло? Что требует действия?

**Шаг 2. Найти подвисшее**
«Подвисшее» — это то, что лежит в блокноте, но требует какого-то решения или действия.

Для каждого подвисшего: либо создать задачу в 📋, либо отправить в «Когда-нибудь».

Третьего не дано. Нельзя оставить в «нигде» — это и создаёт фоновый шум.

**Шаг 3. Обновить раздел «Личное»**
Открыть блокнот текущего месяца и записать:
- Что было на этой неделе (кратко)
- Какие паттерны заметила (в поведении, в работе, в самочувствии)
- Что хочу изменить или попробовать
- Честное «покаяние» — без самобичевания, но без иллюзий

**Шаг 4. Канбан-уборка**
- Всё, что выполнено → перевести в «Готово»
- Всё, что «В работе» но не двигается → вернуть в «План» или в «Когда-нибудь»
- Колонка «Новое» не должна разрастаться — принять решение по каждой задаче

## 7. Совместная работа с AI-агентами

Юля работает с AI-агентом (Эмбер) в Singularity. Это отдельный вид взаимодействия, и здесь есть свои правила.

### Что делает AI, а что — человек

| Задача | Кто делает |
|--------|-----------|
| Создаёт задачи | AI — но всегда предлагает проект и теги для подтверждения |
| Наполняет блокноты | AI — только по запросу, не автоматически |
| Проводит исследования | AI — результат сохраняется в блокнот «Исследования» |
| Принимает решения | Человек |
| Расставляет приоритеты | Человек |
| Сдвигает задачи по канбану | Человек (AI не меняет статус без спроса) |

### Ключевые правила для AI

**AI не сбрасывает статус задач при обновлении.** Если задача «В работе», AI не трогает колонку. Статус меняет только человек.

**Ссылки — через Dropbox, не через markdown.** AI вставляет обычные ссылки, которые можно кликнуть в приложении. Не оборачивает в [текст](ссылка) — это не работает в мобильных интерфейсах.

**После исследования — всегда вопрос:** «Сохранить в заметку?» AI не сохраняет автоматически — ждёт подтверждения. Человек решает, что достойно памяти.

**AI предлагает — человек решает.** Когда AI создаёт задачу, он говорит: «Предлагаю добавить в 📋 Задачи [Homo Divinus] с тегами: Homo Divinus, HD Дети. Подтверждаешь?»

## 8. Реальные примеры: полная структура workspace Юли

Вот как выглядит реальный workspace — с описаниями каждого раздела и подпроекта.

---

### 🌟 Homo Divinus
*Личный бренд Юли: блог, исследования, контент, сайт, подкаст/видео*

- 📋 **Задачи [Homo Divinus]** — все задачи по проекту: контент, сайт, коллаборации, публикации
- 📓 **Исследования** — результаты исследований AI-агента по темам блога. Каждое исследование = заметка с датой и ссылкой на Dropbox
- 📓 **Сайт** — концепция, структура, тексты страниц, технические задачи для разработчика
- 📓 **Кухня с душой** — идеи, рецепты, концепция этого направления контента
- 📓 **Выпуск 1** — материалы, сценарий, заметки по первому эпизоду
- 📓 **Выпуск 2** — аналогично
- 📓 **Архив идей** — идеи для контента, которые не вошли никуда конкретного, но жалко терять

---

### 🤝 Коучинг
*Работа с клиентами, коучинговые программы, инструменты, методология*

- 📋 **Задачи [Коучинг]** — задачи по работе с клиентами, разработка программ, подготовка сессий
- 📓 **Программы** — описания коучинговых программ, их структура, результаты
- 📓 **Инструменты** — рабочие инструменты, упражнения, модели, которые использую в работе
- 📓 **Исследования** — что изучала для профессионального развития

---

### 🏠 Домоведение
*Всё про дом: быт, хозяйство, ремонт, покупки, кухня*

- 📋 **Задачи [Домоведение]** — конкретные бытовые задачи: купить, починить, организовать
- 📓 **Кухня** — рецепты, меню на неделю, идеи блюд
- 📓 **Ремонт / обустройство** — планы, ссылки на товары, решения по интерьеру
- 📓 **Поставщики** — контакты, проверенные мастера, магазины

---

### 📖 Обучение
*Курсы, книги, навыки — всё, что развивает*

- 📋 **Задачи [Обучение]** — записаться на курс, дочитать книгу, применить навык
- 📓 **Книги** — конспекты, цитаты, выводы из прочитанного
- 📓 **Курсы** — материалы, домашние задания, заметки с обучения
- 📓 **Инсайты** — мысли, которые пришли в процессе обучения и стоит сохранить

---

### 🌿 Здоровье
*Тело, врачи, самочувствие, профилактика*

- 📋 **Задачи [Здоровье]** — записи к врачу, анализы, процедуры, спорт

## 9. Чек-лист запуска: как создать такой workspace за 1 час

Возьми чашку чая. Открой Singularity. Вот что делать шаг за шагом.

---

### ⏱️ Минуты 0–15: Определи разделы

1. Возьми листок или открой заметки на телефоне
2. Запиши все сферы своей жизни, которые требуют внимания
3. Объедини похожие
4. Оставь 5–8 крупных разделов
5. Создай их в Singularity как проекты верхнего уровня

**Подсказка:** Не пытайся сделать идеально с первого раза. Разделы можно переименовать и перестроить. Главное — начать.

---

### ⏱️ Минуты 15–30: Создай структуру внутри каждого раздела

Для каждого раздела создай:

1. **Подпроект 📋 Задачи [Раздел]** — назови его именно так, с эмодзи 📋
2. **Подпроект 📓 Исследования** — он будет в каждом разделе
3. **1–2 блокнота** под конкретные темы, которые уже есть в голове

Не нужно создавать все блокноты сразу — они появятся по мере работы.

---

### ⏱️ Минуты 30–45: Настрой теги

1. Открой настройки тегов в Singularity
2. Создай теги для каждого раздела (точь-в-точь как называется раздел)
3. Создай 3–5 тематических тегов для самых активных областей
4. Добавь теги-люди если работаешь не одна
5. Итого: не больше 15–20 тегов

---

### ⏱️ Минуты 45–60: Перенеси то, что уже есть в голове

1. Открой 📋 Задачи в первом разделе
2. Выпиши всё, что крутится в голове по этой теме — без фильтрации
3. Расставь по колонкам: что актуально сейчас → «В работе» или «План», остальное → «Новое» или «Когда-нибудь»
4. Добавь теги
5. Повтори для следующего раздела

**Важно:** Не нужно сразу переносить ВСЁ. Достаточно самого горячего. Система наполняется постепенно.

---

### После запуска: три правила на первую неделю

1. **Каждая новая задача — в нужный 📋**, а не куда попало
2. **Каждая заметка, материал, ссылка — в нужный 📓**, а не в задачу
3. **В пятницу — 30 минут на обзор** (даже если кажется, что нечего смотреть)

После трёх недель система начнёт работать сама. После трёх месяцев — станет второй головой.

## Послесловие

Система Юли — не про идеальный порядок. Она про то, чтобы голова была свободна для думания, а не для запоминания.

Когда всё хранится в правильных местах — появляется ощущение лёгкости. Не «я ничего не делаю», а «я делаю то, что важно, и знаю, что остальное не потеряется».

Попробуй. Начни с малого. Подстрой под себя.

И да — всё, что не работает в твоей жизни, можно изменить. Система служит тебе, а не наоборот.
