## 1. Что такое Цеттелькастен — без академизма

Никлас Луманн — немецкий социолог, написал 70 книг и 400 статей за 40 лет. Секрет: картотека из 90 000 карточек, где каждая идея связана с другими. Не папки с темами, а **сеть мыслей**.

Для Юли это не академический инструмент — это **личный мозг**, который:
- не забывает
- находит связи между идеями
- сам «подсказывает» темы для блога
- копит материал для книги незаметно, день за днём

### Три принципа (адаптированных)

**1. Одна мысль — одна карточка**  
Не «конспект лекции», а «один инсайт из лекции». Если лекция дала 5 мыслей — 5 заметок. Каждая заметка — одна идея, которую можно сформулировать одним предложением.

**2. Ссылки важнее папок**  
Вместо «куда положить эту заметку» — «с чем эта мысль связана?». Заметка про керамику может ссылаться на заметку про медитацию, если там одна суть. Это и есть магия Цеттелькастен.

**3. Своими словами**  
Не копировать чужое — перефразировать. «Автор сказал X, а я думаю Y» — это и есть знание. Цитаты хранить отдельно, ссылки отдельно, а осмысление — в своих заметках.

### Детальный разбор

#### Obsidian
**Плюсы:**
- Файлы хранятся локально на вашем компьютере/телефоне в виде обычных `.md` файлов
- Синхронизация через Dropbox или iCloud — **бесплатно** (у Артура уже настроен Dropbox)
- Никакого vendor lock-in: данные ваши навсегда, даже если Obsidian закроется
- Граф связей — наглядно видно, как идеи соединяются
- Огромный рынок плагинов, включая AI-ассистентов
- Работает в России без VPN

**Минусы:**
- Первоначальная настройка требует ~2-3 часов
- Нужно понять markdown (но он простой: `**жирный**`, `# заголовок`)
- Obsidian Sync стоит $4-8/мес, но **не нужен** при использовании Dropbox

#### Notion
**Статус:** ❌ **Заблокирован в России с 9 сентября 2024 года** в связи с американскими санкциями. Аккаунты российских пользователей отключены. Использование невозможно без постоянного VPN, что ненадёжно для ежедневной системы заметок.

#### Logseq
**Плюсы:**
- Бесплатный, опенсорсный
- Хорош для linked thinking, дневниковых записей
- Файлы тоже локально

**Минусы:**
- Интерфейс блочный (как Roam Research) — непривычен для новичков
- Мобильное приложение нестабильно
- Меньше плагинов и сообщества, чем у Obsidian
- Визуально сложнее

#### Capacities
**Плюсы:**
- Красивый, современный интерфейс
- Встроенный AI
- Типы объектов (заметки, цитаты, люди, книги) — удобно для коуча

**Минусы:**
- Платный за полный функционал (~$9/мес)
- Данные в облаке (не в Европе, неизвестно где именно)
- Нельзя автоматизировать через файловую систему
- Маленькое сообщество, меньше обучающих материалов

#### Вывод по инструментам
**Obsidian** выигрывает для российского пользователя: бесплатный, работает офлайн, синхронизируется через уже существующий Dropbox, открытые файлы позволяют AI-автоматизацию.

## 3. Почему Obsidian — выбор для России

### Синхронизация через Dropbox (бесплатно)

Поскольку Obsidian хранит файлы на диске, достаточно создать папку хранилища внутри Dropbox:

```
~/Dropbox/Юля-Заметки/   ← это хранилище Obsidian
```

Все устройства с установленным Dropbox автоматически синхронизируют заметки. На телефоне — приложение Obsidian для Android/iOS, указывающее на ту же папку через Dropbox-приложение.

**Это работает прямо сейчас, бесплатно, в России.**

### Markdown — это просто

Markdown — это обычный текст с несколькими символами:
- `# Заголовок` — большой заголовок
- `## Подзаголовок` — поменьше
- `**жирный**` — жирный текст
- `- пункт` — список
- `[[Название заметки]]` — ссылка на другую заметку (главная магия Obsidian)

Выглядит обычным текстом, читается без программы. Ваши заметки проживут вечно.

## 4. Структура системы для Юли

### Принцип: минимум папок, максимум тегов

Папки — для типов заметок. Теги — для тем. Это важно: Цеттелькастен работает через связи, не через иерархию.

### Структура папок

```
📂 00-Входящие/          ← сюда падает всё новое (от AI, из Telegram, наспех)
📂 01-Заметки/           ← основные атомарные заметки
📂 02-Конспекты/         ← лекции, курсы, книги (как источники)
📂 03-Цитаты/            ← дословные цитаты с источником
📂 04-Лайки/             ← материалы, которые зацепили (ссылки, статьи)
📂 05-Блог/              ← идеи для постов, черновики, готовые тексты
📂 06-Проекты/           ← активные проекты (Homo Divinus, книга и др.)
📂 07-Шаблоны/           ← шаблоны заметок (не трогать)
```

**Правило:** всё новое падает в `00-Входящие`. Разбор входящих — раз в неделю, 15-20 минут.

### Теги по темам Юли

```
#homo-divinus
#коучинг
#домоведение
#вера
#керамика
#дети
#личное
#психология
#тело
#творчество
```

Один тег = одна тема. Заметка может иметь несколько тегов. Например, инсайт про домашние ритуалы может получить `#домоведение #вера`.

### Специальные теги для типа контента

```
#инсайт        ← личное озарение
#идея-для-блога ← готово к публикации почти
#цитата        ← дословная чужая мысль
#вопрос        ← незакрытый вопрос, нужно исследовать
#книга         ← к книге Homo Divinus или другой
```

### Связи между заметками `[[Wikilinks]]`

Это главная магия. Когда пишете заметку и упоминаете другую — ставьте `[[Название той заметки]]`. Obsidian покажет граф: визуальную карту вашего мышления.

Пример:
> В заметке «Ритуалы утра» вы пишете: «Это перекликается с тем, что я писала в [[Молчание как практика]] и [[Зачем нам нужен порог дня]]»

Через полгода у вас будет живая карта идей, из которой сами будут вырастать главы книги.

## 5. Шаблоны заметок

Шаблоны хранятся в папке `07-Шаблоны/`. В Obsidian есть встроенный плагин Templates — вставляет шаблон в новую заметку одним нажатием.

### Шаблон: Инсайт

```markdown
---
тип: инсайт
дата: {{date}}
теги: [#инсайт, ]
---

# {{title}}

## Мысль
(Одно-два предложения — суть инсайта. Своими словами.)

## Откуда это пришло
(Контекст: после чего возникло — разговор, лекция, книга, прогулка)

## Почему это важно для меня
(Личная связь. Как это меняет мой взгляд?)

## Связано с
- [[]]
- [[]]

## Может лечь в блог?
[ ] Да  [ ] Нет  [ ] Может быть
```

## 6. Как не бросить через неделю

Большинство людей бросают системы заметок из-за **перфекционизма** и **избыточной сложности**. Вот противоядие:

### Правило минимального усилия

**«Лучше плохая заметка, чем никакой заметки»**

Написала одно предложение? Уже хорошо. Шаблон заполнен на 30%? Нормально. Заметка попала не в ту папку? Не страшно — переложить можно потом.

### Ритуал «Входящие»

Один раз в неделю — 15-20 минут — просматриваете папку `00-Входящие`:
- Перемещаете в нужную папку
- Добавляете теги
- Проставляете ссылки на связанные заметки
- Самое важное — превращаете в атомарные заметки

Это ваш «еженедельный обзор» — маленький, но регулярный.

### Три момента захвата

| Момент | Что делать |
|---|---|
| **Читаете что-то интересное** | Сразу в `00-Входящие` — хотя бы ссылку с одним предложением «почему зацепило» |
| **Пришла мысль в голову** | Голосовая заметка в телефон → вечером в систему |
| **После лекции/курса** | 10 минут конспекта сразу, пока свежо |

### Не нужно делать всё идеально

Цеттелькастен — **не архив, а рабочий инструмент**. Если заметка лежит и её никто не читает — она всё равно полезна: вы её думали и писали, это уже переработка информации.

### Маркер прогресса

Раз в месяц открывайте граф связей в Obsidian. Видеть, как растёт сеть идей — мощная мотивация продолжать.

### Не копировать, а думать

Самая частая ошибка — копировать чужой текст. Это не работает. Правило Цеттелькастен: **записывать только то, что поняли**. Если не можете перефразировать своими словами — не поняли ещё, нужно перечитать.

## 7. Автоматизация с AI-ассистентом

Поскольку Obsidian хранит обычные файлы Markdown на диске/Dropbox, AI-ассистент (Эмбер) может **напрямую создавать заметки** без всяких API.

### Как это работает

```
Юля говорит Эмберу: «Сохрани это в мои заметки»
              ↓
Эмбер создаёт файл в ~/Dropbox/Юля-Заметки/00-Входящие/
              ↓
Dropbox синхронизирует на все устройства
              ↓
Юля видит заметку в Obsidian на телефоне/компьютере
```

### Что Эмбер может делать автоматически

**1. Создавать заметки из разговора**
> «Эмбер, запиши инсайт: я поняла, что...»
→ Эмбер создаёт файл по шаблону инсайта в `00-Входящие/`

**2. Сохранять ссылки с контекстом**
> «Эмбер, сохрани эту ссылку — она мне понравилась потому что...»
→ Файл по шаблону ссылки в `04-Лайки/`

**3. Делать рассылки и подборки**
> Каждую неделю Эмбер может создавать файл-подборку с материалами по темам Юли

**4. Готовить черновики для блога**
> «Эмбер, собери из моих заметок пост про [тему]»
→ Эмбер находит связанные заметки → создаёт черновик в `05-Блог/`

### Технически

Путь к хранилищу: `~/Dropbox/openclaw-workspace/` (или отдельная папка для заметок Юли)

Эмбер создаёт файлы через инструмент `write` — напрямую в папку Dropbox. Синхронизация происходит автоматически.

### Telegram-бот как интерфейс

Сценарий: Юля пишет Эмберу в Telegram → Эмбер создаёт заметку → заметка появляется в Obsidian на всех устройствах.

Это **уже работает** — Эмбер имеет доступ к файловой системе и Dropbox.

### Примеры команд для Эмбера

```
«Эмбер, сохрани в заметки: [текст]»
«Запиши инсайт про [тему]»  
«Добавь лайк: [ссылка] — понравилось потому что [почему]»
«Создай черновик поста из моих заметок про домоведение»
«Покажи, что у меня есть по теме Homo Divinus»
```

**День 3-7:**
5. Написать 3-5 первых заметок — любых, просто чтобы привыкнуть
6. Попробовать ссылки `[[]]` между заметками
7. Открыть граф связей (кнопка в левом меню)

**Через неделю:**
Настроить Эмбера на автоматическое создание заметок из Telegram.

**Capacities** — более красивый и интуитивный, встроенный AI, платный (~$9/мес). Минус: нельзя автоматизировать через файловую систему. Подходит как «план Б».
