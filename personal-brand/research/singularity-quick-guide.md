## 1. Структура workspace

```
Раздел (область жизни)
  └── 📋 Задачи [Раздел]   parentOrder=0, ВСЕГДА первый
  └── 📓 Блокнот 1          isNotebook=true
  └── 📓 Блокнот 2
        └── Заметка 1
        └── Заметка 2
```

**Два типа подпроектов — и только два:**
- `📋 Задачи [Раздел]` — канбан-доска. Сюда идут задачи.
- `📓 Блокнот` — хранилище знаний. Заметки, ссылки, материалы. Не задачи.

**Канбан (состояния задач):**
```
Новое → План → В работе → Готово
+ Когда-нибудь (отдельная категория, не колонка)
```

**Приоритеты (числа — инвертированы!):**
- `0` = высокий 🔴
- `1` = обычный
- `2` = низкий 🔵

## 4. Правила для AI-агентов (чек-лист)

### При создании задачи
- [ ] `projectId` — ID подпроекта 📋
- [ ] `group` — taskGroup целевого проекта (см. секцию про API ниже)
- [ ] Эмодзи в заголовке (📝 🩺 💰 🎓 🏕 💎 📷 🎨 📄 🌱)
- [ ] Теги: 1 основной (раздел) + 1–2 дополнительных
- [ ] Чек-лист — для любой нетривиальной задачи
- [ ] Ссылки — прямой URL (Dropbox), не markdown `[текст](url)`

### При обновлении задачи
- [ ] **НЕ менять `state`/`checked`** — статус задачи меняет только человек
- [ ] Не трогать колонку канбана без явного запроса

### После исследования
- [ ] Сохранить результат в 📓 Исследования (или 📓 Исследования AI-lab) в разделе Артефакты
- [ ] Добавить тег темы
- [ ] Спросить пользователя: «Сохранить в Артефакты?» — не сохранять молча

### Три обязательных скилла агента
1. **«Хочешь лучше?»** — предложить улучшение структуры задачи, добавить чек-лист
2. **«Запустить исследование?»** — перед новым проектом предложить сбор данных
3. **«Вот что дальше»** — после выполнения назвать следующий логичный шаг

## 5. Техническое: API и TaskGroup

### Критично: поле `group`

Без корректного `group` задача **не видна в UI приложения** (но существует в API).

```bash
# Получить taskGroup для проекта
GET /task-group?projectId=<projectId>
# → найти объект с parent == projectId → его id = нужный group
```

```json
// Создание задачи — минимально необходимые поля
{
  "title": "📝 Название задачи",
  "projectId": "P-xxxxxxxx",
  "group": "TG-xxxxxxxx"
}
```

**При переносе задачи в другой проект:** обновить оба поля — `projectId` и `group`.

### Чек-лист через API
```bash
POST /checklist-item
{
  "title": "Пункт",
  "parent": "T-xxx",
  "parentOrder": 100
}
# parentOrder: 100, 200, 300... — задаёт порядок
```

### Заметки (Delta/Quill формат)
```bash
# Использовать /note/, НЕ поле note в задаче
PATCH /note/N-T-xxx
{
  "content": "[{\"insert\":\"текст\\n\"}]"
}
# Последний insert ОБЯЗАН заканчиваться на \n
```

### Скрипт
```bash
scripts/singularity.sh raw GET /task-group?projectId=P-xxx
scripts/singularity.sh raw POST /task '{"title":"...","projectId":"P-xxx","group":"TG-xxx"}'
```

**Токены:** Артур → `~/.openclaw/secrets/singularity-api-token`, Юля → `singularity-api-token-yulia`

## 6. Как настроить (для Артура)

### Что нужно
- Singularity App (iOS / Android / web)
- Тариф Elite (299₽/мес) — нужен для API и расширенных функций
- Токен API: Settings → Integrations → API Token

### Создать workspace с нуля

1. **Определить разделы** (5–8 областей жизни + Артефакты)
2. **Создать разделы** как проекты верхнего уровня
3. **В каждом разделе создать:**
   - Подпроект `📋 Задачи [Раздел]` — первым
   - Блокноты под конкретные темы (`isNotebook=true`)
4. **Артефакты:** создать блокноты `Исследования`, `Гайды`, `ТЗ`
5. **Теги:** создать по списку из секции 3 (не больше 20–25)
6. **Синхронизация:** автоматическая через облако Singularity

### Синхронизация с агентами
- Агент Юли (Эмбер) использует токен Юли — видит её проекты
- Агент Артура (Кузя) использует токен Артура — видит его проекты
- Shared-проекты через API **не видны** — только через приложение
- Для семейных задач: дублировать в личные проекты или использовать приложение
