# 🎮 Twine — гайд для быстрого старта

> Twine = бесплатный инструмент для интерактивных историй.
> Результат = HTML-файл, открывается на любом телефоне.
> Сайт: https://twinery.org

## Установка

**Вариант 1 — Онлайн (быстрее всего):**
Открой https://twinery.org/2 — работает прямо в браузере, ничего ставить не нужно. Данные сохраняются в localStorage браузера.

**Вариант 2 — Десктоп (надёжнее):**
Скачай с https://twinery.org → Install → Mac/Win/Linux. Всё сохраняется локально.

## Первые 5 минут

### 1. Создай историю
- Нажми **+ New** (или + Story)
- Назови: «Witchcraft Note — Крыша»

### 2. Открой стартовый пассаж
- Двойной клик на блок **«Untitled Passage»**
- Переименуй в **«Начало»**
- Напиши текст:

```
Ты стоишь у заброшки. Пятиэтажка без окон, стены в граффити. Наверху — крыша.

Лестница — ржавая, но выдержит.

[[Полезть наверх]]
[[Обойти здание]]
```

### 3. Понял принцип?
`[[текст ссылки]]` — создаёт новый пассаж и ссылку на него. Кликнул → попал в новую сцену.

### 4. Добавь вторую сцену
Автоматически появятся два новых блока. Открой «Полезть наверх»:

```
Лестница скрипит. Третий этаж. Пятый. Крыша.

На краю сидит человек. Повязка на голове, борода, гитара за спиной. Рядом — кружка с чаем. Он смотрит на город.

Тебя заметил. Не удивился.

— Чай будешь?

[[Буду.]]
[[А ты кто?]]
[[Молча сесть рядом]]
```

### 5. Запусти
Нажми **▶ Play** (или Build → Play) — откроется твоя игра в браузере.

## Ключевые приёмы

### Ветвление
```
[[Вариант А]]
[[Вариант Б]]
[[Вариант В]]
```
Каждый `[[...]]` = новый пассаж. Игрок выбирает, куда идти.

### Ссылка с другим текстом
```
[[Подойти к нему->Встреча с Арчи]]
```
Игрок видит «Подойти к нему», но попадает в пассаж «Встреча с Арчи».

### Переменные (имя игрока)
```
(set: $name to "Kit")
Тебя зовут $name.
```

### Условия
```
(if: $hasTea is true)[Арчи улыбнулся. — Хороший выбор.]
(else:)[Арчи пожал плечами.]
```

### Картинки (рисунки Ефрема!)
Положи картинку рядом с HTML-файлом (или в папку `images/`):
```
<img src="images/kit.png" width="300">
```
Или base64, если хочешь всё в одном файле.

### Стилизация (тёмная тема)
В пассаже с тегом `stylesheet`:
```css
tw-story {
  background: #1a1a2e;
  color: #e0e0e0;
  font-family: 'Courier New', monospace;
}
tw-link {
  color: #ff6b35;
}
```

## Форматы историй (Story Format)

Twine поддерживает разные «движки»:

| Формат | Стиль | Для чего |
|--------|-------|----------|
| **Harlowe** (по умолчанию) | Простой, минимум кода | Для начала — идеально |
| **SugarCube** | Больше возможностей, инвентарь, сохранения | Если нужна «настоящая» игра |
| **Chapbook** | Самый чистый текст | Для литературных историй |

**Начни с Harlowe** — переключишься позже если нужно.

## Структура проекта (наша)

```
witchcraft-note/
├── README.md          — основной документ проекта
├── docs/
│   └── twine-guide.md — этот гайд
├── art/               — рисунки (Ефрема + AI-концепты)
├── lore/              — персонажи, мир, истории
└── prototypes/        — HTML-файлы с прототипами Twine
```

## Workflow к субботе

1. **Ты** пишешь сцену в Twine (онлайн или десктоп)
2. Экспорт: **Build → Publish to File** → получаешь HTML
3. Сохрани в `prototypes/` (например `v01-roof.html`)
4. Отправь мне — я посмотрю, предложу правки
5. Ефрем открывает на телефоне → видит игру

## Первая сцена — что писать?

Бери «Костёр на поляне» из полевой сессии:
- Kit голодный, бродит по городу
- Находит заброшку, лезет на крышу
- Там Арчи — пьёт чай, бренькает на гитаре
- Разговор → Kit ведёт Арчи на поляну со звёздами
- Костёр, хлеб, колбаска
- Давид может появиться в конце («Кто-то шуршит в кустах...»)

Выборы: за Kit'а. Что он скажет, куда пойдёт, доверится ли Арчи.
