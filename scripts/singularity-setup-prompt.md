# Промпт для настройки Singularity App

## Задача

Настроить Singularity App через REST API: удалить дефолтный мусор, создать структуру проектов.

## Технические детали

**API Base:** `https://api.singularity-app.com/v2`
**Auth:** `Authorization: Bearer <token>` (токен в `~/.openclaw/secrets/singularity-api-token`)
**Формат:** JSON, Content-Type: application/json

### Эндпоинты

| Метод | URL | Что делает |
|-------|-----|-----------|
| GET | /v2/project | Список проектов |
| POST | /v2/project | Создать проект |
| PATCH | /v2/project/{id} | Обновить проект |
| DELETE | /v2/project/{id} | Удалить проект |
| GET | /v2/task?status=active | Активные задачи |
| GET | /v2/task?projectId={id} | Задачи проекта |
| POST | /v2/task | Создать задачу |
| DELETE | /v2/task/{id} | Удалить задачу |
| GET | /v2/tag | Список тегов |
| POST | /v2/tag | Создать тег |

### Создание проекта — тело запроса
```json
{
  "title": "Название",
  "emoji": "1f3e0",
  "color": "#4caf50"
}
```

Emoji — unicode codepoint без `U+` (например: 1f3e0 = 🏠, 1f4bc = 💼, 1f375 = 🍵)

### Создание задачи — тело запроса
```json
{
  "title": "Название задачи",
  "projectId": "P-xxxx",
  "start": "2026-03-16",
  "priority": 0,
  "note": "Описание (markdown)"
}
```
Priority: 0 = нет, 1 = низкий, 2 = средний, 3 = высокий

## Промпт для агента

```
Ты настраиваешь Singularity App через REST API для семьи Трофимовых.

Токен: прочитай из ~/.openclaw/secrets/singularity-api-token
API: https://api.singularity-app.com/v2
Auth header: Authorization: Bearer <token>

### Шаг 1: Посмотри что есть
- GET /v2/project → выведи все проекты с ID
- GET /v2/task?status=active → посчитай задачи, покажи первые 20

### Шаг 2: Удали мусор
Удали проект "Добро пожаловать!" (если есть) — DELETE /v2/project/{id}
Удали все задачи, которые выглядят как обучающие от Singularity (содержат слова: "Загляните", "Познакомьтесь", "Нажмите на стрелку", "Добавьте задачи в секцию", "База знаний — подробная справка", "техник управления временем", "Составьте список дел", "Загляните в настройки", "планирование: тем, кому нужно", "Аврал на работе", "заставить себя работать", "Перенесите задачи из «Входящих»", "Дела-перерывы").

Для каждой задачи, которую удаляешь — сначала покажи её название и спроси подтверждение (выведи список, жди "да").

### Шаг 3: Оставь существующие проекты если полезны
Проекты которые уже есть и могут остаться:
- 💼 Работа → переименовать в "Jiffy" если хочет
- 🏡 Личное → может стать "Дом" 
- 📚 Учёба → может стать "Развитие"
- 🍵 Чайная миссия → оставить
- Ефрем → может войти в "Семья" или остаться отдельным
- Завершение дел в Ха → оставить как есть

Спроси перед переименованием.

### Шаг 4: Создай недостающие проекты
Создай те, которых нет:

| Название | Emoji (codepoint) | Цвет |
|----------|-------------------|------|
| Семья | 1f46a | #e91e63 |
| Вера | 26ea | #9c27b0 |
| Финансы | 1f4b0 | #ff9800 |
| Кухня | 1f37d | #8bc34a |
| Witchcraft Note | 1f3ae | #673ab7 |
| Тело | 1f94b | #f44336 |

### Шаг 5: Создай полезные теги
POST /v2/tag для каждого:
- срочно
- ждёт ответа  
- с Юлей
- еженедельно
- покупка

### Шаг 6: Покажи итог
Выведи финальный список проектов и количество задач в каждом.

ВАЖНО:
- Перед удалением — показать что удаляешь и ждать подтверждения
- Перед переименованием — спросить
- Не трогать задачи которые выглядят как реальные (например "Договориться о деловой встрече с Ларисой", "Сформировать следующий оптовый заказ")
- Rate limit: делай паузу 0.5 сек между запросами (sleep 0.5)
```
