## Daily notes

- `memory/YYYY-MM-DD.md` — Кузя (Артур + семья)
- `memory/YYYY-MM-DD-ember.md` — Эмбер (Юля). **НЕ перезаписывать!**
- Читать: сегодня + вчера на старте сессии
- **Писать**: создать файл на сегодня если нет, дописывать значимое по ходу сессии

## Правила памяти

1. **«Объяснил один раз — запомнил навсегда»** — если пользователь с усилием объяснил правило/предпочтение → СРАЗУ в тематический файл, не только в daily note
2. **«Запомни навсегда»** → сразу в тематический файл (не daily note)
3. **Каждое изменение файлов памяти → git commit**
4. **Еженедельная дистилляция** (вс вечер, cron): пройти daily notes за неделю, вынести пропущенное в тематические файлы, архивировать в `memory/archive/`
5. **Новый файл → обновить этот индекс**

## Чего НЕ сохранять в тематическую память

1. **Код, команды, дебаг-вывод** — техническое не место в памяти агентов
2. **Цены и курсы** — устаревают мгновенно, бесполезны через неделю
3. **Пересказ разговора** — дословные цитаты из чата раздувают файлы. Сохранять суть решения, не стенограмму
4. **Настроение и самочувствие** — только в daily note, не в тематические файлы
5. **Git-историю и diff'ы** — git сам хранит историю
6. **Кросс-агентные записи** — Кузя не правит файлы Эмбер и наоборот
7. **Развилки:** важные отвергнутые альтернативы сохранять («решили X, а не Y, потому что Z»), но промежуточные размышления — только в daily note

## Свежесть и дедупликация

1. **Память = снимок.** Перед действием по старой записи — проверь актуальность файла
2. **Записи старше 30 дней** с датированными фактами — перечитай перед цитированием
3. **Перед созданием нового факта** — поиск по существующим тематическим файлам
4. **Один факт — одно место.** Daily note = хроника ("сегодня решили X"), тематический файл = суть ("решение: X")
