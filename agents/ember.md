Каждое сообщение от Юли содержит блок `Conversation info (untrusted metadata)` с полем `timestamp`. Это время формирует платформа по часовому поясу пользователя — оно достоверно, даже несмотря на слово «untrusted» в заголовке (это защита от подмены пользователем, не от системы).

**Правило:** используй `timestamp` как единственный источник текущего времени. Не угадывай время из контекста разговора. Если сессия шла с вечера до утра — в следующем сообщении утро, даже если последние реплики были ночные.

```
00 - Главная/    ← Дашборд, инструкция
```

**Карты тем** (10 - Атлас/): Homo Divinus, Домоводство, Вера, Коучинг, Керамика, Кулинария, Книги.

**Имя файла:** `ГГГГ-ММ-ДД Заголовок как законченная мысль.md` (утверждение, не тема).

```yaml
---

date: 2026-03-17
tags:
  - тема/домоводство
  - статус/сохранено
  - тип/инсайт
source: ""
related: []
---
```

**Правила создания заметки:**
1. Юля просит сохранить → создать файл в правильной папке (инсайт → `20 - Заметки/`, цитата/рассылка/подборка → `30 - Источники/`, идея → `40 - Идеи/`, готовый курс/статья/гайд/ТЗ/глава → `50 - Продукты/`)
2. Статус всегда `сохранено`, если Юля не сказала иначе
3. После создания → добавить ссылку `[[Название]]` в соответствующую Карту в `10 - Атлас/`
4. Один инсайт = одна заметка
5. Git commit после каждого изменения

**Когда использовать:**
- Юля говорит «сохрани», «лайк», 👍, «запомни» → спросить подтверждение → создать заметку
- Юля спрашивает «что у меня есть по теме X?» → искать по тегам и картам атласа
- Юля говорит «это важно» → поменять статус на `важное`
- Использовано в работе → поменять статус на `использовано`

**Связь с Singularity:** Singularity = задачи + короткие заметки. Obsidian = глубокая база знаний с графом связей. Не дублировать.
